================================================================
                    用户端 API 使用文档
================================================================

Base URL：https://api.987ai.vip
请求格式：Content-Type: application/json
鉴权方式：无需全局认证头，凭证以请求体字段传入
文档日期：2026-05-15

----------------------------------------------------------------
                         接口总览
----------------------------------------------------------------

 #  | 方法   | 路径                          | 功能                     | 限流
----|--------|-------------------------------|--------------------------|------------------
 1  | GET    | /api/card-keys/:cardCode       | 查询卡密状态（含库存）   | 30次/分钟/IP
 2  | POST   | /api/parse-token               | 解析 Access Token        | 20次/分钟/IP
 3  | POST   | /api/tasks                     | 创建充值任务             | 10次/分钟/IP
 4  | GET    | /api/tasks/:taskId             | 查询单个任务状态         | 60次/分钟/IP
 5  | DELETE | /api/tasks/:taskId             | 取消排队中的任务         | 20次/分钟/IP
 6  | POST   | /api/tasks/batch               | 批量查询任务状态         | 30次/分钟/IP
 7  | POST   | /api/card-keys/batch-query     | 批量查询卡密状态         | 10次/分钟/IP
 8  | POST   | /api/check-account             | 查询账号订阅信息         | 无限流
 9  | POST   | /api/card-keys/replace         | 旧卡密换新卡密           | 5次/分钟/IP


================================================================
                       详细接口说明
================================================================

----------------------------------------------------------------
1. 查询卡密状态（含库存信息）
----------------------------------------------------------------
GET /api/card-keys/:cardCode

说明：
  此接口同时返回卡密可用状态与库存级别信息。
  库存查询已整合至本接口，无需再单独调用 /api/stock-info（已废弃）。
  只要卡密存在且绑定了平台账号，无论卡密是否可用，均会尝试返回库存信息。

路径参数：
  cardCode  string  卡密（大写字母+数字）

响应示例：
  可用（含库存）：
  {
    "available": true,
    "stock_level": "high"
  }

  不可用（同样携带库存信息）：
  {
    "available": false,
    "error": "卡密已被使用",
    "stock_level": "low"
  }

  卡密不存在（无库存信息）：
  {
    "available": false,
    "error": "卡密不存在"
  }

stock_level 取值：
  "high"  库存充足（> 100）
  "low"   库存偏少（1 ~ 100）
  "none"  库存为空
  ""      平台账号未开启库存展示，或卡密未绑定平台账号

error 取值：
  "卡密不存在" / "卡密已被使用" / "卡密已停用"


----------------------------------------------------------------
2. 解析 Access Token
----------------------------------------------------------------
POST /api/parse-token

请求体：
  {
    "access_token": "<ChatGPT Access Token>"
  }

响应示例：
  成功：
  {
    "success": true,
    "message": "user@example.com"
  }

  失败：
  {
    "success": false,
    "message": "Token无效或已过期"
  }

注意：成功时 message 字段返回的是账号邮箱（脱敏处理）。


----------------------------------------------------------------
3. 创建充值任务
----------------------------------------------------------------
POST /api/tasks

请求体：
  {
    "card_key": "ABCD1234",
    "access_token": "<Access Token 或 UUID 格式的 app_user_id>",
    "idp": "",
    "force_recharge": false
  }

字段说明：
  card_key        string  是  卡密
  access_token    string  是  ChatGPT Access Token；
                              也接受 UUID 格式的 app_user_id 直接使用
  idp             string  否  登录方式标识，可为空字符串
  force_recharge  bool    否  是否强制充值（忽略账号类型校验）

响应示例：
  成功：
  {
    "success": true,
    "task_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  }

  失败：
  {
    "success": false,
    "error": "卡密不可用"
  }

  已有任务进行中（HTTP 409）：
  {
    "success": false,
    "error": "该卡密有任务正在处理中，任务ID: <uuid>，请等待处理完成"
  }

注意：409 冲突时，可从 error 字段正则提取 task_id 继续轮询。


----------------------------------------------------------------
4. 查询单个任务状态
----------------------------------------------------------------
GET /api/tasks/:taskId

路径参数：
  taskId  string  任务 UUID

响应示例：
  {
    "status": "pending",
    "result": "",
    "error": "",
    "queue_position": 3
  }

status 取值：
  pending     排队等待中
  processing  处理中
  completed   已完成
  failed      失败
  unknown     任务不存在

注意：
  queue_position 仅在 pending 状态时有值（1 表示下一个处理）。
  建议轮询间隔：3 秒。


----------------------------------------------------------------
5. 取消排队中的任务
----------------------------------------------------------------
DELETE /api/tasks/:taskId

路径参数：
  taskId  string  任务 UUID

响应示例：
  成功：
  {
    "success": true
  }

  失败：
  {
    "success": false,
    "error": "任务不在排队状态，无法取消"
  }


----------------------------------------------------------------
6. 批量查询任务状态
----------------------------------------------------------------
POST /api/tasks/batch

请求体：
  {
    "task_ids": ["uuid1", "uuid2", "..."]
  }

限制：单次最多 50 个任务 ID。

响应示例：
  {
    "results": {
      "uuid1": {
        "status": "completed",
        "result": "...",
        "error": "",
        "queue_position": 0
      },
      "uuid2": {
        "status": "pending",
        "result": "",
        "error": "",
        "queue_position": 2
      }
    }
  }


----------------------------------------------------------------
7. 批量查询卡密状态
----------------------------------------------------------------
POST /api/card-keys/batch-query

请求体：
  {
    "card_keys": [
      "ABCD1234",
      "https://987ai.vip/recharge?code=EFGH5678",
      "订单号: IJKL9012"
    ]
  }

限制：
  单次最多 500 条
  每条输入长度不超过 500 字符
  支持格式：
    - 原始卡密（如 ABCD1234）
    - 充值链接（自动提取 code/card 参数）
    - 带分隔符格式（自动解析最后一段卡密）

响应示例：
  {
    "success": true,
    "message": "查询成功",
    "data": [
      {
        "card_key": "ABCD1234",
        "status": 0,
        "status_text": "正常",
        "link": "https://987ai.vip/recharge?code=ABCD1234",
        "user_id": "",
        "used_at": ""
      },
      {
        "card_key": "EFGH5678",
        "status": 1,
        "status_text": "已使用",
        "link": "https://987ai.vip/recharge?code=EFGH5678",
        "user_id": "abcdef@gmail.com",
        "used_at": "2025-01-01 12:00:00"
      }
    ]
  }

status 取值：
   0   正常      可用（其他状态统一展示为正常，不返回 user_id / used_at）
   1   已使用    返回 user_id（原始值，不脱敏）和 used_at（北京时间）
   2   已停用    返回 user_id（原始值，不脱敏）和 used_at（北京时间）
  -1   无效      无法从输入提取有效卡密
  -2   不存在    卡密在数据库中不存在

  }


----------------------------------------------------------------
9. 自助换卡
----------------------------------------------------------------
POST /api/card-keys/replace

请求体：
  {
    "old_card_key": "ABCD1234"
  }

注意：字段自动转换为大写并去除首尾空格。

响应示例：
  成功：
  {
    "success": true,
    "message": "换卡成功",
    "data": {
      "new_card_key": "NEWKEY9999"
    }
  }

  失败：
  {
    "success": false,
    "message": "卡密已被使用，无法更换"
  }

失败场景一览：
  HTTP 400  无效的请求参数          参数缺失
  HTTP 400  卡密格式无效            格式不符合规范
  HTTP 400  卡密不存在              数据库中无此卡密
  HTTP 400  卡密已被使用，无法更换  状态为已使用
  HTTP 400  卡密已停用，无法更换    状态为已停用
  HTTP 409  卡密有任务正在处理中    存在进行中的任务，附带任务 ID
  HTTP 500  生成新卡密失败，请稍后重试  服务器内部错误


================================================================
                     任务失败错误说明
================================================================

任务失败时（status: "failed"），error 字段经服务端脱敏处理：

  错误场景                    返回内容
  --------------------------  --------------------------------------------------
  账号已是目标会员类型        充值失败，用户已为XXX，请更换账户（原文返回）
  库存不足                    无可用库存，请稍后再试
  其他所有错误                若提交多次还是失败，请打开浏览器的无痕模式重新
                              登陆gpt，获取最新的密钥
